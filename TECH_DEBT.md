# ТЕХНИЧЕСКОЕ ЗАДАНИЕ И АРХИТЕКТУРНЫЙ ДОЛГ БОТА

## 1. Назначение
Разработать HFT асинхронного торгового бота для **Phemex Futures**. Лимитная логика входа, динамическая лимитная логика выхода ("снайпер"), контроль по стакану.
Бот находит точку входа по паттерну стакана, открывается лимиткой, сразу переходит в режим динамического хантинга цели в стакане. Работает устойчиво при частичных наливах, отменах и реконнектах.

**ГЛАВНОЕ ПРАВИЛО:** Как только от биржи пришло подтверждение о частичном исполнении ордера (вход/выход) — остаток немедленно отменяется.

## 2. Основной режим и Конфигурация
* Биржа: Phemex USDT Futures.
* Мультиактивность: `app.max_active_positions >= 1` (параллельная торговля).
* Хедж-мод (концептуальный): Допускается торговля Long и Short по одному символу, если позволяет конфиг, строго изолированно.

## 3. Инварианты
1. По одной монете-стороне не может быть двух циклов.
2. Входы и выходы — строго лимитные.
3. Остатки после `PartiallyFilled` отменяются фоново.
4. После ручного закрытия позы оператором: стейт сбрасывается, монета свободна, штрафов нет.
5. Идемпотентность WS: дубли не ломают логику.
6. Отказ API: если бот в позиции, стейт **не удаляется** ни при каких лимитах ошибок. Бот переходит в Экстрим, а при его отказе — зовет оператора.

## 4. Жизненный цикл выхода (Exit Lifecycle)

### 4.1. Average Scenario (Сердце стратегии)
Бот вычисляет виртуальный Тейк Профит на базе `ask2/bid2` момента сигнала. 
Динамический хантинг: бот сканирует нужную часть стакана (биды для лонга, аски для шорта). Если находит цену `(>= / <= virtual_tp)` с наибольшим объемом — кидает туда лимитку. Любой выкупленный объем фиксируется, остаток отменяется.
Через `shift_ttl` снижает планку на `shift_demotion`. При достижении `min_target_rate` + таймаут — переход в Экстрим.

### 4.2. Negative Scenario
Если после входа спред находится в негативной зоне (хуже или равен входу) на протяжении `negative_ttl`, позиция сбрасывается в Экстрим.

### 4.3. Interference (Скупка помех)
Скупка мелких объемов (в рамках average_vol_pct_to_init_size), мешающих дойти до основной цели. Купленный объем просто плюсуется к общему балансу позиции. На саму виртуальную цель это никак не влияет. Поскольку система работает без физических зависших тейк-профитов, AverageScenario на следующем тике стакана автоматически учтет новый увеличенный объем при поиске ликвидности для выхода. В лонговом сценарии помехи ищутся среди асков. Для шортового — среди бидов. Область для сканирования: > entry_price и < target_price.

### 4.4. Position TTL
Жесткий таймер. Если время вышло, цель переводится на точку входа (+ `to_entry_orientation`). При неисполнении — Экстрим.

### 4.5. Extrime Close
Экстренный выход. Бот рассчитывает ось `mid = (ask1 + bid1) / 2` и ставит лимитку со смещением `shift`, прогрессивно ухудшая цену (пробивая стакан) каждые `retry_ttl` секунд, гарантируя маркет-подобное исполнение с защитой лимита.

## 5. Recovery (Восстановление)
При старте грузит локальный стейт -> сверяет с биржей -> оставляет только то, что есть на бирже. Забытые локальные позиции удаляются, чужие биржевые позиции игнорируются (никакого удочерения).
