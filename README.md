# Rucheiok Trading Bot (Phemex Futures)

Асинхронный высокочастотный скальпер для фьючерсов Phemex. Реализует паттерн-стратегию "Ручеек" с глубокой вложенной системой сценариев управления позицией (Average, Negative, Interference).

## ⚙️ Архитектурные принципы
- **Strict Limit Execution:** В системе нет Market ордеров. Все сделки, включая экстренные выходы, исполняются агрессивными лимитными заявками для защиты от проскальзывания.
- **Stateless Engines:** Движки (`ENTRY`, `EXIT`) не хранят стейт. Они принимают слепок стакана и отдают чистые математические команды (`action`).
- **Event-Driven Sync:** Вход по WSS стакана, мутация стейта — по приватному WSS ордеров. Внедрена динамическая синхронизация `pos.qty` с реальным балансом биржи (для защиты от частичных закрытий).
- **Atomic Recovery:** Весь рантайм-кеш атомарно дампится в `bot_state.json`. При рестарте математика скрещивается с реальными позициями биржи.

2. Настроить `.env`:
   ```env

Утилита принудительной расстановки плечей: python tools/set_global_leverage.py

🛡 Инварианты конфигурации (cfg.json)
app.max_active_positions: Лимит открытых сделок. Защищен синхронным локом от гонки потоков.

quota_asset: Строго "USDT". Бот автоматически клеит его к Blacklist.

risk.leverage: Поддерживает числа (20), строку "max" (максимум по спецификации биржи), или null (дефолт аккаунта).

risk.notional_limit: (USDT). Если объем входа или помехи превышает этот лимит, бот жестко обрезает ордер.

risk.quarantine: При достижении max_consecutive_fails монета улетает в бан на quarantine_hours. Успешная сделка сбрасывает счетчик в ноль. "inf" заносит в Blacklist навсегда.

Вложенность EXIT: Все блоки обязаны присутствовать в cfg.json, даже если их enable установлен в false.

🧬 Жизненный цикл алгоритма (Pipeline)
1. ENTRY (Вход)
EntryEngine находит паттерн. Бот бронирует слот в памяти, капит объем по notional_limit и кидает Limit Order.

Приватный WSS ловит PartiallyFilled (моментальная отмена остатка) или Filled. Регистрация init_qty в памяти.

2. EXIT (Закрытие)
Первичная цель: После stabilization_ttl выставляется Take Profit.

Average: По таймеру (shift_ttl) цель сдвигается ближе к точке входа (ухудшается).

Negative: Если спред держится в минусе negative_ttl -> запуск Extrime Close.

Position TTL: По истечении position_ttl цель переносится в безубыток (entry_price). Если через breakeven_wait_sec сделка не закрыта -> Extrime Close.

3. INTERFERENCE (Скупка помех)
Ищет крупные уровни (от avg_vol_pct) СТРОГО в коридоре между entry_price и текущим Take Profit.

Объем докупок ограничен max_vol_pct от стартового init_qty.

При выкупе помехи бот плюсует объем и переставляет TP ровно на ту же закешированную цену, сохраняя прогресс сдвига Average.

4. EXTRIME CLOSE (Экстренный выход)
Блокирует все остальные проверки.

Кидает агрессивную лимитку в центр спреда, ухудшая цену в сторону невыгодного стакана.

При исчерпании retry_num кидает глубокую лимитку (2% за спред) для гарантированного рыночного забора ликвидности без риска отбоя Market-ордера.
